# 程序员跳槽全攻略

- 规划(选项/持续/理性) vs 计划

## 跳槽

- 原理
  - 价值论: 人才的使用价值(业务能力)-供需(毕业季/细分市场)-信息透明度(选择题vs判断题)-交换价值-雇主(薪资)
  - 跳槽不是什么
    - 价值最大化: 工作是生活的一部分 职业的不可逆性
    - 找工作vs换工作 -> 衡量得失别裸辞
    - 加薪 -> 个人成长+能力提升->发展空间+业务挑战->匹配的薪资
  - 为什么跳槽
    - 求职本质: 工作最重要的目的--积累金钱
    - 意义: 市场需求与自我实现匹配 职位和能力匹配
    - 原则: 现在很差vs未来更好
- 准备
  - 自我目标和自我实现
  - 职业四象限: E-雇员(专家/管理) B-创业 S-自由 I-投资
  - 路线图: 节点-达标能力/参考时间 风险最小
  - 市场需求: 定量分析->定制市场分析->定性分析
  - 根据需求调整定位: 技术趋势 + 投资新兴市场和细分市场
    - 新机会: 帮助你成长 汇报对象 福利 位置(城市) 职业策略(安定->事业) 工作生活平衡 兴趣点 组织文化 合作者 公司背景
    - 时薪+机会成本
  - jobDeer 职业画布
    - 架构价值: 我是谁 我有什么资源 我的竞争优势 谁可以帮助我
    - 传递价值: 雇主需要什么样的人 怎么让雇主知道你 怎么宣传和证明自己
    - 预估收益
  - 个人品牌: github blog 微博 技术社区账号(sf) 量化
  - 沟通和写作: 条理性 逻辑性
  - 分享: 源码+渠道
  - 开源: 技术转型
  - 架构: DRY 半自动化 正交性
- 操作
  - 简历: 本质 -> 约到面试; F-feature A-advantage B-benefit; 论据vs论题; 对比体现成长
  - 工具+模板
  - 人脉(内推): 最优途径
  - 猎头(注意隐私保护)
  - 直投: boss/快简历
  - 知识补全计划: 简历中的 技能树
  - 常见面试题 - 面经
    - 离职: 未来更好
    - 挑战: 压力->太多, 讲一个最近的吧
    - 三年后: 成长稳定与发展空间
    - 反问: 兴趣 对产品的好奇
    - 缺点: 生活上的缺点工作上的优点 -- 较真说话直
    - 加班: 救急
    - 行为面试法
  - 知己知彼 准时和礼节
  - 离职 离职访谈: 未来更好

## 求职

- 自我介绍: 5分钟版本 1-2分钟版本
- 行业: 不讨厌 有期待; 平台->习惯/约束; 大公司/小公司->能力提升/责任与自由

职场

- 选择努力同样重要
- 领导让你上就上
- 不要过早离开一线
- 只全栈不纵深要不得
- 趁早打造影响力
- 精通一门外语
- 基础还要扎实
- 好好保重身体
